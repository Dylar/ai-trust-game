# Network Package

This package contains small HTTP transport helpers shared across service handlers.

Its purpose is to keep transport concerns consistent without moving business decisions into generic helpers.

## What Lives Here

- [`request.go`](./request.go)
  request middleware that attaches request metadata to the context

- [`metadata.go`](./metadata.go)
  metadata type plus context helpers

- [`response.go`](./response.go)
  shared JSON response writer

## Request Metadata

[`RequestMiddleware`](./request.go) adds a `Metadata` value to every wrapped request context.

Current fields:

- `RequestID`
  generated by the server for each request

- `SessionID`
  read from `X-Session-Id`

- `UserID`
  read from `X-User-Id`

The middleware also writes `X-Request-Id` back to the response so the request can be correlated in logs or tests.

## Why This Matters

The metadata package keeps request-scoped technical context explicit.

That is especially important for flows such as:

- interaction processing, which loads authoritative session state from `SessionID`
- audit events, which attach request and session identifiers from the current context
- HTTP logging, which benefits from a stable request ID at the service edge

This package does not decide whether a request is allowed.
It only transports metadata so the service layer can make explicit decisions with it.

## JSON Responses

[`WriteJSON`](./response.go) centralizes basic JSON responses for handlers.

It currently:

- sets `Content-Type: application/json`
- writes the response status
- encodes the provided payload when one exists

This keeps simple handler responses consistent across the service layer.
